# McSandwich

## Description

- From TCSS 460 assignment. This is a sub sandwich restaurant website which has four main pages: Home, menu, Order and Cart page. The frontend is mainly based on Bootstrap.
- The users can sign in and create a new account to order foods. The food options are customizable and orders will be saved to user account.
- The backend is provided by a Node.js web services, with PostgreSQL as database. This backend handles all authentication and order operations.
- For API documentation: <https://hungvu-mcsandwich.herokuapp.com/api/>

## Features

- Home page: A simple business introduction, (menu) carousel and testimonials.

- Menu page: A static page to presents a sample menu of the day.

- Order page:
  - Has 2 tabs: Customized order and quick order. Quick order is like an order preset.
  - For customized order (focus of "Order"), it displays a running tally (total price) in real time, and give users all option to create their own version of sub sandwich.
  - User can name, reset and add order to cart.
  - After the user add order to cart, the user must click on "Go to cart" or a cart icon on top right to go to Shopping cart".

- Shopping cart page:
  - A user can see order detail by click on order name.
  - A user can see price of each order.
  - A user can remove single order.
  - A user can remove multiples orders at once.
  - A user can rename multiple orders at once.
  - A user can update/alter order.
  - A user can add "Favorite" order (only for logged in user).
  - A user can clear cart.
  - A user can place order (only for logged in user).
  - A user can see total price of the cart.
  - When click on update order icon (a gear). The user will navigate back to order page. The order page will be populated by the info of that order. The user can press "Add to cart" again to save the order.
  - When proceed to place order, the user can review orders. The user can click on order name to see the detailed information. From here, the user can complete ordering or go back.

- Profile page:
  - Located in a dropdown menu (hover over username on top right), or "base_url/profile". The page is secure and only authenticated user can access.
  - Used to see and modify previous orders.

- Sign in:
  - The user can sign in. All fields must not be empty.
  - There is no restriction on user name.
  - The password requires at least 8 character, with at least 1 upper case letter.
  - After signing in, the username will be located on top right corner, and it will be there even when you navigate to different pages.
  - After signing in, the username can sign out.

- Registration:
  - The user can registration. All fields must not be empty.
  - Password use the same rule as sign in, there is no restriction on other fields.
  - Password and Retype password must match.
  - After finish registration, the user is directly signed in.

- Some more information about backend:

  - Authentication:
    - Client-side and server-side verification for inputs (no blank fields, and some specific rules for different inputs). For password, it needs at least 8 characters, with at least 1 uppercase letter.
    - Password is stored with Salt/Hashing algorithm in database.
    - Registration requires first, last name, username, email, password, retype password.
    - JWT is provided in a HTTP cookie.
    - Sign out.

  - Previous Order:
    - GET previous order feature is implemented.
    - Secure enpoint, only authenticated users can access.
    - This can be seen in a profile page.
    - The previous order can be deleted by clicking on a "X" icon next to each other.

  - Side notes:
    - Postman test script for API endpoints is included, however, some test might not work unless some specific information is manually adjusted to create a specific test condition/flow. They are indicated via test name, and most are related to cookies.
